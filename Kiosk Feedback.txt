사용 할때 객체로 선언을 받을 것.

lateinit = 무조건 생성자에서 초기화 해줘야 함. 
          = 나중에 초기화 하겠다. 대신에 이 경우에서 초기화 안할 경우 컴파일 안해줌. 
          = 지금은 내가 메모리에 안오리지만, 다음에 내가 사용할거임. 을 의미함.

ex) 

lateinit var pay: Payment

fun main() {
	pay = Payment()
}



~Logic 함수가 필요 없었다. 그리고 꼬리물기 하면 댐.




===== 그냥 추가 피드백 =====
1. 변수명 / 함수명 / 파일명 제대로 짤 것. 목적에 맞게

2. 클래스 다이어그램을 짜고 나서, 역으로 생각해보는 것 
	예시) 어떤 기능을 내가 유지보수 하려고 찾는 가정

3. 중복코드가 보이면, 일단 의심을 해봐야 한다

4. 특정 흐름으로만 프로그램이 동작되도록 개발하면 안됨.
예시) 처음부터 끝까지 꼬리물기를 해놓은것과 다를게 없다.
